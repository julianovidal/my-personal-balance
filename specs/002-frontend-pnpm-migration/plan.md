# Implementation Plan: Frontend pnpm Migration

**Branch**: `002-frontend-pnpm-migration` | **Date**: 2026-04-06 | **Spec**: [spec.md](spec.md)  
**Input**: Feature specification from `/specs/002-frontend-pnpm-migration/spec.md`

## Summary

Migrate the `frontend/` service from npm to pnpm as the Node package manager. This involves replacing `package-lock.json` with `pnpm-lock.yaml`, updating `frontend/Dockerfile` to install and invoke pnpm, updating `frontend/AGENTS.md` to document pnpm for developer and agent context, and verifying the full `podman-compose up --build` workflow succeeds end-to-end.

## Technical Context

**Language/Version**: Node.js 22 (via `node:22-alpine` Docker image); TypeScript 5.9  
**Primary Dependencies**: pnpm (latest stable via `corepack enable pnpm`); existing frontend deps unchanged (React 18, Vite 7, Tailwind 3, TanStack Query 5)  
**Storage**: N/A — no storage layer involved  
**Testing**: Manual end-to-end verification (`pnpm install`, `pnpm run dev`, `podman-compose up --build`)  
**Target Platform**: Linux container (Alpine); local macOS/Linux development  
**Project Type**: Web application frontend (React + Vite SPA)  
**Performance Goals**: N/A — package manager change does not affect runtime performance  
**Constraints**: Must not alter any `package.json` scripts or runtime behaviour; must remain compatible with `podman-compose` orchestration  
**Scale/Scope**: Single directory (`frontend/`); no backend or classifier changes required

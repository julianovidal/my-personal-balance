# Quickstart: Frontend pnpm Migration

## Prerequisites

- **pnpm** installed locally. The recommended way:
  ```bash
  corepack enable pnpm
  ```
  Or via npm (one-time bootstrap only):
  ```bash
  npm install -g pnpm
  ```
- **Node.js 22** (matches the Docker image base)
- **Podman** + **podman-compose** (for container verification)

## Local Development After Migration

```bash
# 1. Remove stale npm node_modules (if present)
rm -rf frontend/node_modules

# 2. Install dependencies
cd frontend
pnpm install

# 3. Start the development server
pnpm run dev
# → App available at http://localhost:5173
```

## Build for Production

```bash
cd frontend
pnpm run build
```

## Container Verification

```bash
# From project root
podman-compose -f podman-compose.yml up --build

# Verify frontend is healthy
curl http://localhost:5173
```

## Smoke Test Checklist

Run these after completing the migration tasks to confirm everything works:

- [x] `pnpm install` completes with zero errors in `frontend/`
- [x] `pnpm-lock.yaml` exists; `package-lock.json` is gone
- [x] `pnpm run dev` starts and app is reachable at `http://localhost:5173`
- [x] `podman-compose up --build` succeeds and all services are healthy
- [x] No `npm` references remain in `frontend/Dockerfile` or `frontend/AGENTS.md`

## Troubleshooting

| Symptom | Cause | Fix |
|---------|-------|-----|
| `pnpm: command not found` | pnpm not installed | Run `corepack enable pnpm` |
| `ERR_PNPM_OUTDATED_LOCKFILE` in Docker | `pnpm-lock.yaml` out of sync with `package.json` | Run `pnpm install` locally and commit updated lockfile |
| Old `node_modules` causes resolution errors | Generated by npm, incompatible structure | Delete `node_modules` and re-run `pnpm install` |
